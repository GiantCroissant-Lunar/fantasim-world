# Spec-First Checklist

Use this checklist before starting any implementation work.

## Pre-Implementation

- [ ] Identified the relevant RFC(s) for this work
- [ ] Read the RFC(s) completely (not just skimmed)
- [ ] Checked for related ADRs
- [ ] Understood the vocabulary (governance vs pipeline)
- [ ] Identified stable contracts/IDs that must not change

## Spec Locations Checked

- [ ] `../fantasim-hub/docs/rfcs/v2/` (primary)
- [ ] `../fantasim-hub/docs/rfcs/v2/RFC-INDEX.md` (spine)
- [ ] `../fantasim-hub/docs/adrs/` (decisions)
- [ ] `../fantasim-hub/docs/TERMINOLOGY.md` (vocabulary)

## Understanding Verified

- [ ] Can explain the design in my own words
- [ ] Know which components are truth vs derived
- [ ] Know the persistence requirements (MessagePack/RocksDB)
- [ ] Identified any spec gaps or ambiguities

## Ready to Implement

- [ ] Spec summary documented
- [ ] Key constraints listed
- [ ] Interfaces identified
- [ ] Open questions raised with user
